## Проблема:

Ни один компьютер в сети не может получить доступ к HTTP серверу и к DNS серверу. Задача:

1. Найти решение проблемы доступа компьютеров к серверам
2. Настроить пароли на всех коммутаторах для: консольные линии, линии vty, привилигерованный доступ и сделать баннер с текстом (All configs are defended by the law)

## Гипотезы решения
1) Отправлял пакеты ICMP на сервера из подсети 10.0.30.0/24 в подсеть 10.0.120.0/24
2. Рассмотрел настройки статического маршрута на всех роутерах
3) В настройках DHCP не было DNS сервера

## Решение
1) Настроил на каждом роутере статические маршруты
```
ip route 10.0.120.0 255.255.255.0 10.0.3.1 -- из подс сети (10.0.120.0) с серверами по подсети роутера (10.0.3.1)
```
2) Дописал DHCP на всех роутерах для всех подсетей кроме 10.0.120.0/24
3) Настроил на коммутаторах пароли на привелигированный режим, линии vty, на консольный линии, а также поставил баннер.Все пароли зашифрованы.
```
line console 0
password <password>   --- консольные линии
login 

line vty 0 15
password <password>   --- линии vty
login

enable secret <password>   --- привелигированный режим

banner motd #<message>#   --- баннер

service password-encryption   --- шифрование паролей
```
